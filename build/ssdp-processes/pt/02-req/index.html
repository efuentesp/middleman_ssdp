Desenvolvimento dos Requisitos

Consiste em entender os processos e objetivos de negócio do cliente, adquirir o conhecimento de negócio e desenvolver a especificação dos requisitos funcionais e não funcionais da aplicação (nova ou a ser modificada). Inclui as atividades de: reunir, entender, analisar, especificar e validar os requisitos.

Diagrama do Proceso


Descrição do Proceso

Entradas Passos Saídas
  • PRP. Proposta ou Solicitação de Trabalho (SOW)
  • BPM. Modelo de Processos de Negócio (como está e como deverá estar) [opcional].
  1. Inspecionar a Proposta / Solicitação de Trabalho
  2. Coletar e entender os requisitos (negócio, funcionais, não funcionais) com os stakeholders.
  3. Gerar o Glossário de Negócio.
  4. Especificar o contexto do sistema (limites e fronteiras
  5. Gerar a estrutura de dados / modelo de domínio.
  6. Gerar o modelo de Casos de Uso / Serviços.
  7. Planejar as tarefas de especificação de requisitos.
  8. Gerar as especificações dos requisitos funcionais (casos de uso, regras de negócio, modelo de domínio, layouts de interface e notas de implementação).
  9. Gerar as especificações dos requisitos não funcionais.
  10. Inspecionar as especificações dos requisitos.
  11. Validar as especificações dos requisitos com o cliente.
  12. Estabelecer uma Baseline dos requisitos e um processo de administração de mudanças.
  13. Post-mortem REQ.
  • BRD. Documento de Requisitos de Negócio
  • FRS. Especificação de Requisitos Funcionais:
    • Controle de Requisitos
    • Diagrama de Contexto
    • Modelo de Casos de Uso / Serviço
    • Especificações de Casos de Uso / Serviço
    • Especificação de Regras de Negócio e/ou Tabelas de Decisão
    • Modelo de Domínio / Modelo Lógico Entidade-Relacionamento
    • Glossário de Negócio
    • Dicionário de Dados
    • Modelo UML
    • Inventário de mensagens
  • NFR. Especificação de Requisitos Não Funcionais.
  • COM. Especificação de Características Comuns da Aplicação.
