Integración del Sistema

Es el proceso de integración de componentes funcionales en un sistema completo (sistema construido) y la prueba de interacciones directas entre componentes funcionales (llamadas, invocaciones e intercambio de mensajes). Si existen interfaces con sistemas externos, se deben probar tales interacciones. Si la plataforma de desarrollo es distinta a la plataforma de producción, se debe probar la compatibilidad entre ellas. El proceso de integración de componentes incluye actividades para detectar y corregir defectos de integración/interacción entre componentes.

Diagrama de Proceso


Descripción del Proceso

Entradas Pasos Salidas
  • ABS. Estructura de descomposición de la aplicación y dependencia entre componentes.
  • FCSI. Especificaciones detalladas de componentes (para componentes a integrar).
  • FCI. Componentes funcionales implementados (para componentes a integrar).
  1. Elegir integración intercomponentes a ser probados (interacciones directas entre componentes funcionales e interfaces con sistemas externos)
  2. Identificar compatibilidad/portabilidad de plataformas y escenarios a ser probados
  3. Planear tareas de integración
  4. Diseñar alcance de las pruebas de integración/portabilidad, escenarios y datos
  5. Preparar integración/portabilidad en ambiente de pruebas
  6. Inspeccionar conectividad de plataformas y técnica
  7. Instalar y configurar aplicación
  8. Ejecutar casos de prueba de integración/portabilidad
  9. Generar manual de deployment/instalación con la configuración de ambiente y aplicación
  10. Promover aplicación al ambiente de pruebas de sistemas (pruebas de ciclo de negocio y desempeño)
  11. Post-mortem INT
  • APP. Aplicación integrada (paquete para deployment).
  • DMI. Manual de deployment/instalación.
  • ILOG. Evidencia de ejecución de pruebas de integración y log de defectos.
