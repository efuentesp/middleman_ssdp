PDSS v3 Tradicional

O Processo de Desenvolvimento de Software da Softtek (SSDP) divide-se em 6 fases sequencias distribuídas no tempo, cada uma conta com um milestone principal, cada fase é essencialmente um período de tempo que ocorre entre dois milestone principais. Ao final de cada fase é feita uma valorização para determinar se os objetivos foram alcançados. Uma valorização satisfatória permite seguir em frente para a próxima fase. Cada fase está composta por um conjunto de atividades e cada produto é atribuído a um conjunto de regras específicas.

Processo de Desenvolvimento

Nome Descrição
ENG. Compromisso Esta fase faz parte do ciclo de vendas. Encarrega-se de transformar os requisitos do cliente em uma solução proposta, de acordo com as informações fornecidas e que seja viável para a operação, respeitando o modelo selecionado de Desenvolvimento de Aplicações e todos os elementos deste conjunto.
REQ. Desenvolvimento dos Requisitos Neste processo cria-se uma Baseline dos artefatos que darão suporte para o entendimento dos objetos e processos de negócio do cliente, adquirindo o conhecimento do negócio e desenvolvimento das especificações dos requisitos funcionais e não funcionais da aplicação (nova ou a ser modificada). Este deverá ser aderente e coerente com o escopo original do projeto definido na Declaração de Trabalho (SOW) / Proposta.
ARC. Desenho e Implementação da Arquitetura Neste processo serão tomadas as decisões, além de definir a plataforma que será baseada a aplicação. Com o resultado desta tomada de decisões técnicas, se implementará tanto o framework como os subsistemas de suporte necessários para satisfazer os requisitos funcionais e não funcionais.
SFD. Desenho Funcional da Solução Define e desenha uma solução específica para os requisitos funcionais e não funcionais aprovados do projeto.
DBI. Desenho e Implementação da Base de Dados Transforma o Modelo Lógico da Base de Dados em um modelo físico capaz de satisfazer os requisitos de acesso a dados, volumes de dados esperados, desempenho, concorrência, integridade e privacidade dos dados exigidos dos componentes funcionais.
CIS. Especificação e Implementação dos Componentes É o processo de integrar as considerações técnicas dentro das especificações dos componentes funcionais.
