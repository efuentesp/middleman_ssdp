Desenho dos Testes do Sistema

É o processo de definição da estratégia de testes e geração de testware de sistemas a verificar: •A correta execução do processo de negócio de início ao fim para os cenários de operação esperados (ciclos de negócio completos); •Desempenho aceitável para sobrecarregar o sistema sob limites operacionais esperados (concorrência, o volume de dados e frequência de transações).

Diagrama do Proceso


Descrição do Proceso

Entradas Passos Saídas
  • FCD. Estrutura de decomposição da aplicação e dependência entre componentes;
  • FCS. Especificação detalhada de componentes;
  • DBI. Base de dados implementada;
  • NFR. Requisitos de volume e desempenho estimado;
  • ILD. Scripts de carregamento de BD com conteúdo de catálogos e dados iniciais (fornecidos pelo cliente).
  1. Identificar cenários de ciclo de negócio/variantes;
  2. Validar cenários de ciclo de negócio/variantes (com o cliente);
  3. Planejar tarefas de desenho de testes de ciclo de negócio;
  4. Desenhar scripts de testes de ciclo de negócio e calcular resultados esperados;
  5. Definir o conjunto de dados para testes de ciclo de negócio (sincronizados com sistemas externos);
  6. Inspecionar o desenho de testes de ciclo de negócio;
  7. Preparar o ambiente de testes de ciclo de negócio;
  8. Selecionar os componentes críticos para testes de desempenho (transacional e batch);
  9. Selecionar recursos potencialmente restringidos (elementos de dados, tabelas e etc.);
  10. Selecionar estruturas de dados de alto volume;
  11. Desenhar os cenários de testes de tempo de resposta (para componentes que necessitam de alto desempenho com acesso a estruturas de alto volume);
  12. Desenhar os cenários de teste de concorrência (para recursos potencialmente restringidos, interfaces com sistemas externos e concorrência entre processos bacth e linha);
  13. Validar os cenários de teste de desempenho e critérios de aceitação (com o cliente);
  14. Planejar as tarefas de desenho de testes de desempenho;
  15. Gerar scripts de teste automatizados (usando ferramentas de teste de desempenho/estresse);
  16. Definir conjunto de dados de teste de desempenho;
  17. Preparar o ambiente de teste de desempenho;
  18. Post-mortem STD
  • STW. Testware de sistemas (cenários de teste de ciclo de negócio e cenários de teste de desempenho).
