Diseño Funcional de la Solución

Es el proceso de definir/diseñar una solución específica a los requerimientos funcionales del proyecto dentro de Especificaciones Funcionales y No-Funcionales. La solución consiste de un conjunto de componentes funcionales integrados en un sistema, así como las interacciones entre los componentes y actores externos (usuarios u otros sistemas), y sus fuentes de datos (bases de datos o sistemas de archivos). Las especificaciones funcionales serán complementadas con modelos de base de datos lógicos y prototipos de interfaz de usuario (wireframes). Cada componente funcional debe apegarse a un patrón funcional definido en el proceso ARC. Las especificaciones funcionales deben apegarse a estándares y lineamientos para describir las actividades internas que deben llevarse a cabo en cada interacción. Las especificaciones deben describir solo el comportamiento externo y las actividades internas, dejando las decisiones de la estructura interna de los componentes a las disciplinas de diseño técnico.

Diagrama de Proceso


Descripción del Proceso

Entradas Pasos Salidas
  • FRS. Especificación de Requerimientos Funcionales .
  • FPS. Especificaciones de Patrones Funcionales (autorizados para su uso).
  • UIS. Estándares de Interfaz de Usuario.
  1. Identificar los componentes funcionales y su trazabilidad hacia los CU y reglas de negocio.
  2. Generar una estructura de descomposición de la aplicación y dependencias entre componentes funcionales.
  3. Planear las tareas de especificación funcional de la solución.
  4. Diseñar el modelo lógico de la base de datos y datos globales de la aplicación (constantes, variables, enumeraciones y contadores.
  5. Diseñar el comportamiento de la interfaz de usuario y prototipo para los componentes línea.
  6. Diseñar interfaces de interacción con otros sistemas y layouts para los componentes fuera de línea.
  7. Especificar el comportamiento funcional externo y las actividades internas.
  8. Inspeccionar las especificaciones funcionales.
  9. Validar el diseño funcional de la solución (con el cliente).
  10. Establecer la línea base del diseño funcional y del proceso de administración de cambios.
  11. Post-mortem SFD.
  • ABS. Estructura de descomposición de la aplicación (lista de componentes funcionales agrupados en módulos).
  • FCD. Dependencia de componentes funcionales de la aplicación.
  • FCS. Especificaciones de los componentes funcionales (comportamiento externo y actividades internas).
  • DOM. Modelo lógico entidad-relación/Modelo de Dominio (actualizado).
  • EIS. Layouts de Interface (Especificación de interfaz Externa)
  • LOFI. Prototipo de GUI baja fidelidad.
  • CFG. Datos Globales de la aplicación (constantes, variables, enumeraciones y contadores).

Roles

Nombre Descripción
PL. Líder de Proyecto Planear Actividades. Validar alcance del producto y autorizar el presupuesto del proyecto.
BA. Analista de Negocio Encabezar las actividades de diseño funcional de la solución y producir entregables.
SA. Arquitecto de Software Proveer de patrones funcionales y de estándares de interfaz de usuario.
SD. Diseñador de Software. Validar entregables (precisión y apego a estándares).
STH. Stakeholder. Validar entregables.
