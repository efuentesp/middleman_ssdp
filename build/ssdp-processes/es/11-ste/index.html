Ejecución de las Pruebas de Sistema

El proceso de pruebas de integración de aplicaciones valida la precisión de la prueba de ciclo de negocio de punta-a-punta y que se hayan satisfecho los requerimientos de desempeño en el ambiente de producción (o similar) bajo condiciones de estrés. El proceso de ejecución de pruebas de sistemas incluye las actividades para la detección y corrección de defectos encontrados.

Diagrama de Proceso


Descripción del Proceso

Entradas Pasos Salidas
  • STW. Testware de sistemas:
  • APP. Aplicación integrada (paquete para deployment).
  1. Planear tareas ejecución de pruebas de ciclo de negocio y desempeño
  2. Inspeccionar preparación de ambiente de ciclo de negocio
  3. Ejecutar casos de prueba de ciclo de negocio, verificar resultados y registrar defectos. Los casos de prueba a ejecutar han considerado la validación de los datos de entrada y el grado de seguridad de los datos
  4. Reparar defectos encontrados (si existen) y regresar al paso 3
  5. Inspeccionar preparación del ambiente de desempeño
  6. Instalar y configurar herramientas de monitoreo y medición de desempeño
  7. Ejecutar scripts de prueba de desempeño, registrar métricas y analizar resultados. Los casos de prueba a ejecutar han considerado la validación de los datos de entrada y el grado de seguridad de los datos
  8. Inspeccionar y optimizer acceso a base de datos implementados en la aplicación [solo si es necesario]
  9. Optimizar configuración de base de datos y servidores (particionamiento, memoria, hilos de procesamiento, pool de conexiones, etc) [solo si es necesario]
  10. Evaluar si la aplicación puede promoverse al ambiente de pruebas de aceptación
  11. Promover la aplicación al ambiente de pruebas de aceptación
  12. Post-mortem STE
  • APP. Aplicación con pruebas de sistema.
  • SLOG. Evidencia de ejecución de pruebas de sistema y log de defectos.
