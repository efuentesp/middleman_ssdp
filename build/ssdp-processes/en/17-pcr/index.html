Project Change Request

Process that manages changes in project scope according to the last Baseline defined and agreed with the client. Evaluates the impact of the changes identified and negotiating them with the client, authorized changes are incorporated into the work plan and they establish a new baseline for the project.

Process Diagram


Process Description

Inputs Steps Outputs
  • PRP. Proposal or Statement of Work (SOW)
  • TRM.Traceability Matrix
  • DEL. Deliverable Product list
  • BAS. Actual Project Baseline
  • MWP. Master Work Plan (updated)
  • PDP. Project Development Plan (Client's Change Management Procedures and Standards)
  • CHR. Client's Change Request
  1. Record Change Request.
  2. Analyze and get change evidence.
  3. Document Change Request (Software Product, Process or Baselined Product )
  4. Notify Customer & Project Sponsor about PCR.
  5. Present Product Scope Change Request to Project Sponsor.
  6. Analyze Resource Impact (time, cost, people, etc.)
  7. Present Product Scope Change Request to Project Sponsor.
  8. Update baseline for Project Plan or Process or software product.
  9. Update Contract.
  10. Execute Request with Resources provided by Client.
  • PCR. Project Change Request
  • MWP. Master Work Plan (updated)
  • PCRL. Project Change Requests Log

Roles

Name Description
PL. Project Leader He/She is the person responsible together with his/her team to establish the strategy and the project planning to meet established commitments with the customer
BRM. Business Relationship Manager Person who is responsible for dealing with customers in relation to the sale of Softtek services.
DM. Delivery Manager Person who is responsible to give an explanation to directors of all the projects assigned to him.
CCB. ​Change Controls Board They are responsible for approving changes to the base line and determine acceptance or rejection of a requested change in terms of their impact on the project.
