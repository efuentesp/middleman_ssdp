Diseño e Implementación de la Base de Datos

Es el proceso de transformar el modelo de datos lógico a un modelo físico de base de datos capaz de satisfacer las peticiones acceso a los datos desde los componentes funcionales, almacenar los volúmenes de datos esperados, satisfacer los requerimientos de desempeño y concurrencia, y preservar la integridad de la información almacenada y privacidad.

Diagrama de Proceso


Descripción del Proceso

Entradas Pasos Salidas
  • DOM. Modelo Lógico (Entidad-Relación).
  • NFR. Especificación de requerimientos no funcionales (estimados de volumen y de desempeño).
  • DBS. Estándares de Base de datos [del cliente y de Softtek].
  • ARS. Especificación de arquitectura de aplicación. Modelos generales de la arquitectura. [especificación de solución a los requerimientos no funcionales].
  • SEC. Estándares de seguridad SSecPro.
  • FCS. Especificación de los componentes funcionales [opcional, para complementar la inspección y para la optimización de accesos].
  1. Inspeccionar el modelo de estructura de datos – soporta la persistencia y solicitudes de acceso a los datos descritos en las especificaciones de componentes funcionales
  2. Identificar los estándares, lineamientos y restricciones para la implementación física de la base de datos (provistos por el cliente
  3. Generar el modelo lógico de la base de datos y define reglas de integridad y restricciones (llaves primarias, llaves foráneas y restricciones de dominio)
  4. Normalizar el modelo lógico de la base de datos (3ra Forma Normal) e incluir estructuras de datos redundantes (de-normalización y consolidación de repositorios) para la optimización del desempeño
  5. Identificar columnas de control para administración de concurrencia y mecanismo de bitácora (timestamps, registro de versiones, usuarios que crearon y actualizaron)
  6. Diseñar la estructura de la base de datos (tablas, columnas, atributos de las columnas) cumpliendo con las convenciones de nombres y estándares.
  7. Diseñar el panorama de instancias de la base de datos (instancias para desarrollo, pruebas y producción; localización requerida, espacios y configuración).
  8. Implementar las instancias de la base de datos (desarrollo y pruebas).
  9. Diseñar e implementar la seguridad de acceso a la base de datos (considerando los requerimientos de seguridad, la arquitectura, el documento de Administración de la Configuración y los estándares de seguridad de SSecPro).
  10. Identificar e implementar objetos de la base de datos (indices, vistas, restricciones y triggers)
  11. Verificar el modelo de base de datos físico contra los requerimientos de volumen y de desempeño.
  12. Establecer la línea base del modelo de datos físico y proceso de administración de cambios.
  13. Especificar el mapeo entre el modelo de estructura de datos de dominio y el modelo físico de la base de datos (ORM)
  14. Desarrollar y probar la capa de acceso a datos de la aplicación basada en el ORM (Mapeo Objeto Relacional)
  15. Diseñar e implementar procedimientos de almacenamiento y de respaldo/recuperación [opcional].
  16. Post-mortem DBI
  • PDM. Modelo de base de datos físico (DDL).
  • DBI. Instancias de la base de datos de desarrollo y pruebas.
  • ORM. Mapeo objeto relacional (mapeo de la estructura de datos de dominio al modelo de base de datos físico).
  • DAL. Capa de acceso de datos de la aplicación (funciones de acceso a la base de datos o componentes de mapeo objeto-relacional) [Implementados y probados].
