System Test Execution

The application integration testing process validates both, the accuracy in the execution of “end-to-end” business process cycles, and that the performance requirements under stress conditions in the production environment (or similar) be met. System Test Execution process includes the activities for detecting and fixing found defects.

Process Diagram


Process Description

Inputs Steps Outputs
  • STW. System testware:
  • APP. Integrated Application (Deployment Package).
  1. Planning tasks test execution and business-cycle performance
  2. Inspect preparation of business-cycle environment.
  3. Execute business-cycle test cases, confirm results and log defects. Test cases have considered the validation of the input data and the extent of data security.
  4. Fix defects found (if any) and return to step 3.
  5. Inspect performance environment preparation.
  6. Install and configure performance monitoring and measurements tools.
  7. Execute performance test scripts, log metrics and analyze results. Test cases have considered the validation of the input data and the extent of data security.
  8. Inspect and optimize database accesses implemented in the application (queries and SQL statements) [only if required].
  9. Optimize database and servers configuration (partitions, memory, processing threads, connection pool, etc.) [only if required].
  10. Assess readiness for release to acceptance test environment.
  11. Promote application to acceptance test environment.
  12. Post-mortem STE.
  • APP. Application with system testing applied.
  • SLOG. System Testing Execution Log and Defect Log.

Roles
