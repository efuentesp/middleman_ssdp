System Integration

Integration process of functional components into a complete system (system developed) and the direct interactions test among the functional components (calls, invocations and message exchange). If there are interfaces with external systems, the interactions must be tested. If the development platform is different from the production platform, the compatibility between them must be tested. The component integration process includes activities to detect and correct defects of integration/interaction among components.

Process Diagram


Process Description

Inputs Steps Outputs
  • ABS. Application Breakdown Structure (list of functional components grouped in modules)
  • FCSI.Implementation specification of the functional component (addendum to Functional Component Specification).
  • FCI. Functional Components Implemented (for components to integrate).
  1. Choose Integration among components to be tested (direct interactions among functional components and interfaces with external systems).
  2. Identify compatibility/portability of platforms and scenarios to be tested.
  3. Plan Integration
  4. Design scope of integration testing/portability, scenarios and data.
  5. Prepare integration and portability in test environment
  6. Inspect platforms and technical connectivity.
  7. Install and configure application
  8. Execute test cases of integration and portability
  9. Generate deployment and installation manual with the configuration of environment and application
  10. Promote application to system test environment(business cycle testing and performance)
  11. Post-mortem INT.
  • APP. Integrated Application (Deployment Package).
  • DMI. Deployment and installation Manual.
  • ILOG. Integration test execution and defects log evidence.
