Construcción de Componentes

Es el proceso de diseñar la estructura interna (identificando y diseñando componentes técnicos), codificar y probar a profundidad un componente funcional.

Diagrama de Proceso


Descripción del Proceso

Entradas Pasos Salidas
  • IMPG. Lineamientos de implementación (Convenciones de Nombrado, Estándares técnicos y de codificación, Restricciones etc.).
  • ARCI. Implementación del framework de la aplicación.
  • DBI. Instancias de la Base de Datos de Desarrollo y Pruebas.
  • FCSI. Especificación de implementación del componente funcional.
  • ARS. Especificación de Arquitectura de Aplicación.
  • GUI. Prototipo IU (HTML, Formas).
  1. Inspeccionar la especificación funcional del componente
  2. Planear tareas de construcción de componente funcional
  3. Diseñar estructura interna del componente
  4. Diseñar y preparar la prueba funcional
  5. Codificar componente
  6. Compilar código
  7. Ejecutar prueba funcional
  8. Evaluar liberación al ambiente de integración
  9. Promover componente funcional al ambiente de integración
  10. Post-mortem CTR
  • FCI. Componentes funcionales implementados (diseñados, codificados, probados funcionalmente y liberados al ambiente de pruebas).
    • Diseño de la Estructura interna
    • Código Fuente (uno o varios archivos)
    • Script de Compilación (job o proyecto IDE)
    • Script para Deployment (job o proyecto IDE)
    • Testware unitario automatizado (p.ej JUnit o NUnit)
    • Testware funcional (script y datos)
    • Evidencia de ejecución de pruebas funcionales y log de defectos
    • Documentación autogenerada (p.ej. JavaDocs)

Roles

Nombre Descripción
SE. Ingeniero de Software Encabezar las actividades de construcción y producir entregables.
TE. Ingeniero de Pruebas Diseñar y preparar testware y ejecutar pruebas independientes.
SD. Diseñador de Software Proveer de entradas y validar entregables.
TL. Lider Técnico Revisar las actividades de construcción y entregables generados.
